# Proof packet: workspace lifecycle state machine

Purpose: anchor proof that Aureline owns a single canonical workspace lifecycle
state machine for readiness labels (especially `partially_ready`) and that the
live shell path surfaces those states honestly while remaining interactive for
basic editing.

Canonical sources (non-exhaustive):

- `docs/workspace/lifecycle_state_machine.md`
- `crates/aureline-workspace/src/lifecycle/mod.rs`
- `crates/aureline-shell/src/bootstrap/native_shell.rs`
- `crates/aureline-shell/src/palette/query_session.rs`
- `fixtures/workspace/lifecycle_cases/`

Evidence storage:

- Validation captures: `artifacts/milestones/m1/captures/`
- Smoke outputs (optional): `artifacts/milestones/m1/smoke_outputs/`
- Screenshots (optional): `artifacts/milestones/m1/screenshots/`
- Runtime logs (generated by the shell): `.logs/workspace/`

Suggested manual drill (protected walk):

- Start the shell from a workspace directory: `cargo run -p aureline-shell --bin aureline_shell`
- Execute `Open Folder` (current prototype uses the current working directory).
- Observe the status bar readiness badge transitioning:
  - `Partially Ready` while the file index is scanning / watcher is warming
  - `Ready` once the file index completes and watcher health is `healthy`
- Trigger a degraded case:
  - Toggle trust to restricted via the shell toggle (`Cmd/Ctrl+Shift+T`) and
    confirm the readiness badge reports `Degraded` until trust is restored.
