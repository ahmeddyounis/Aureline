# M5 Storage, Retention, Export, and Offboarding Matrix for New Durable Artifacts

Checked-in artifact: `artifacts/release/m5/publish_the_m5_storage_retention_export_and_offboarding_matrix_for_new_durable_artifacts.json`

Human-readable companion: `docs/m5/publish_the_m5_storage_retention_export_and_offboarding_matrix_for_new_durable_artifacts.md`

This matrix governs the storage posture, retention policy, export path, and offboarding behavior for every new durable artifact class in Milestone 5. It is the canonical source for downstream docs, Help/About, CLI inspection, and support export surfaces.

## Artifact classes covered

- `user_owned_local` — personal settings, profiles, local history
- `workspace_owned_managed` — workspace state with optional managed mirror
- `ai_memory` — AI context, embeddings, and generated memory
- `sync_state` — sync-backed state and device registry
- `session_export` — exportable session artifacts
- `derived_cache` — regenerable caches and indexes
