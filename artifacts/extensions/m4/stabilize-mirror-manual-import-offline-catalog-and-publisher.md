# Artifact: Stabilize mirror/manual import, offline catalog, publisher-transfer continuity, and namespace-trust flows

**Task:** Promote the imported / mirrored / offline marketplace row into the stable line — bind the source class (route, visibility, source-class-preserved flag, last-known-good pin, offline-explainable flag), the publisher-transfer continuity (namespace reservation, ownership transfer, signer/key rotation, orphaning/succession, approved-mirror promotion — each with cooldown, audit trail and lineage, user/admin notification, and a high-trust auto-update gate), the mapping outcome generated from the real imported artifact (exact / translated / partial / shimmed / unsupported, with a preserved rollback checkpoint and diagnostics on failure), the permission posture, compatibility, activation budget, and install/revocation/mirror posture into one validated packet, and derive the stability qualification with automatic narrowing below Stable so users, admins, and reviewers can all inspect and trust the row.
**Status:** Implemented
**Verification class:** Conformance / interoperability suite + Security / privacy review + Docs validation + Release evidence review

## Summary

This lane binds, for an imported / mirrored / offline row, the identity (catalog descriptor ref, row identity, package identity, pinned import-truth version, publisher namespace, source artifact ref, publisher trust tier, lifecycle state), the source class (import route `primary_catalog` / `approved_mirror` / `offline_bundle` / `manual_artifact`, source visibility, source-class-preserved flag, last-known-good ref + pin, offline-explainable flag), the publisher-transfer continuity (continuity event `none` / `namespace_reservation` / `ownership_transfer` / `signer_key_rotation` / `orphaning_succession` / `approved_mirror_promotion`, continuity state, cooldown, audit trail + lineage, user/admin notification, high-trust auto-update gate, transfer-history preservation), the mapping outcome (`exact` / `translated` / `partial` / `shimmed` / `unsupported`, generated from the real imported artifact, with a rollback-checkpoint ref + preserved flag and a diagnostics ref), the permission posture (declared / effective refs, widened-on-import flag), the compatibility label, the worst-case activation-budget instrumentation, and the install posture (install scope + disclosure, revocation posture, mirrorability, rollback support) into one validated packet, and derives the stability qualification it may claim. A `stable` import-truth claim is only allowed when the row pins the published import-truth version, is evidence-backed, keeps its publisher trust tier out of quarantine, stays on an installable lifecycle, preserves its source class, stays explainable offline, pins a last-known-good, keeps its publisher-transfer continuity current with a preserved audit lineage and transfer history (and gates high-trust auto-update behind delay / audit / notification for any transfer event), maps exactly or by a verified translation from the real artifact with no missing rollback checkpoint, never widens permissions on import, keeps verified non-inherited non-parity-limited compatibility, keeps its activation cost bounded and within budget, discloses its install scope, keeps a clean revocation posture, stays mirrorable, and is fully attributed. A non-installable lifecycle, a disputed / revoked continuity, an ungated transfer auto-update, an unsupported mapping, a missing rollback checkpoint, a permission widened on import, an unsupported compatibility, an unbounded activation cost, or a quarantined / revoked revocation posture withdraws the row; an in-cooldown / stale continuity, a shimmed / partial mapping, a parity-limited compatibility, an over-budget activation cost, an advisory revocation posture, or a not-mirrorable row narrows to `beta`; an unpublished import version, a catalog-asserted basis, a quarantined trust tier, an unpreserved source class, an unexplainable-offline row, a missing last-known-good, a pending-notification / orphaned / missing continuity, an incomplete audit lineage, an unpreserved transfer history, a missing user/admin notification, a mapping not from the real artifact, an inherited / unverified compatibility, an unmeasured activation cost, an undisclosed install scope, or an incomplete attribution narrows to `preview`. When any condition fails the visible tier is automatically narrowed below Stable with machine-readable reasons. The checked-in packet is canonical: marketplace result / detail rows, install / side-load / mirror-bundle review, the offline catalog view, diagnostics, support exports, the CLI inspector, and release packets ingest it instead of cloning a "Trusted" badge or going dark when a publisher is revoked or rehomed.

## Guardrails honored

- No ambient extension privilege: a permission set widened on import withdraws the row (`widened_permissions_on_import_withdraws_the_row`); `allows_ambient_privilege` is pinned false.
- No catalog-only trust: a `catalog_asserted_only` basis can never back a stable claim (`catalog_asserted_basis_cannot_back_stable`, `no_catalog_only_stable_claim`); `allows_catalog_only_trust` is pinned false.
- No unbounded activation cost: an `unbounded` budget withdraws the row (`unbounded_activation_cost_withdraws_the_row`); `over_budget` narrows to `beta`; `allows_unbounded_activation_cost` is pinned false.
- High-trust auto-update is gated: a transfer event that resumes auto-update before delay / audit / notification withdraws the row (`ungated_auto_update_on_transfer_event_withdraws_the_row`).
- Offline explainability survives revocation/rehoming: the last-known-good pin and source class are preserved and carried into the support/mirror export even for a disputed or withdrawn row.
- A narrower stable claim is published rather than papered over: the effective tier, downgrade flag, reasons, and banner are re-derived from the posture at validation time, so the packet cannot drift.

## Risks / follow-ups

- The continuity event, state, and posture are producer-supplied closed strings; when the publisher-continuity registry exposes typed enums, these should be sourced from that model directly rather than re-declared as strings here.
- The activation budget is summarized as a single worst-case surface posture; a later revision should carry a per-surface activation row so a reviewer can see which surface drove the worst case.
- Trust-tier, lifecycle, compatibility-evidence, activation-budget, stability-tier, and claim-basis vocabularies are closed string sets shared with the catalog-truth, lifecycle-flow, SDK-author-lane, manifest-hardening, runtime-ABI, external-host, and Wasm-host-governance stable lanes; when those crates stabilize typed enums, these should be narrowed to share them rather than re-declared as strings.
- The compatibility record carries a single label + window ref; a later revision should resolve the bound compatibility window's own freshness rather than accepting a producer-supplied `compatibility_label_class`.
- The mapping outcome carries the rollback-checkpoint and diagnostics refs; a later revision should resolve and replay the bound checkpoint to confirm rollback-to-last-known-good rather than accepting a producer-supplied `checkpoint_preserved` flag.
