# Effective Policy, Remembered-Decision, Waiver-Expiry, and Exception-Preview UX — Stabilize Fixtures

Reviewer-facing fixtures for the stable proof packet that promotes the beta
policy simulation page to a verified stable claim. The packet proves that
effective-policy previews, remembered decisions, waiver/exception expiry, and
exception-preview UX links are all verifiably complete and auditably sound.

The canonical record kind is `policy_effective_policy_stabilize_page_record`.
The runtime module lives at
[`/crates/aureline-policy/src/stabilize_effective_policy_remembered_decision_waiver_expiry_and/mod.rs`](../../../../crates/aureline-policy/src/stabilize_effective_policy_remembered_decision_waiver_expiry_and/mod.rs).

## Files

These JSON files are generated by the headless example so they stay a literal
projection of the seeded `EffectivePolicyStabilizePage` (no hand-written drift):

```sh
cargo run -q -p aureline-policy --example dump_effective_policy_stabilize_fixtures -- page > page.json
cargo run -q -p aureline-policy --example dump_effective_policy_stabilize_fixtures -- defects > defects.json
cargo run -q -p aureline-policy --example dump_effective_policy_stabilize_fixtures -- summary > summary.json
cargo run -q -p aureline-policy --example dump_effective_policy_stabilize_fixtures -- support-export > support_export.json
cargo run -q -p aureline-policy --example dump_effective_policy_stabilize_fixtures -- drill-raw-private-material-withdrawn > drill_raw_private_material_withdrawn.json
cargo run -q -p aureline-policy --example dump_effective_policy_stabilize_fixtures -- drill-missing-change-class-preview > drill_missing_change_class_preview.json
cargo run -q -p aureline-policy --example dump_effective_policy_stabilize_fixtures -- drill-exception-preview-links-missing > drill_exception_preview_links_missing.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full stabilize audit page (rows + defects + summary + embedded beta page). |
| `defects.json` | Defect array (empty on the seeded page — all six stability conditions hold). |
| `summary.json` | Aggregate banner (row counts, change classes covered, expiry count). |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `drill_raw_private_material_withdrawn.json` | Drill: raw private material exposed in exception — packet withdrawn. |
| `drill_missing_change_class_preview.json` | Drill: `policy_bundle_change` simulation removed — packet narrowed to preview. |
| `drill_exception_preview_links_missing.json` | Drill: exception preview refs cleared in all simulations — narrowed to beta. |

## Stability conditions verified

1. **Upstream beta page clean** — `policy_simulation_beta_page` audits with zero defects.
2. **Required change classes covered** — `policy_bundle_change` and `settings_lock_change` each have at least one simulation.
3. **Exceptions bounded and attributable** — every exception/waiver has an explicit expiry, named owner, renewal path, and revocation path.
4. **Remembered-decision drift explained** — every drifted, expired, or force-retired decision names at least one typed invalidation reason.
5. **Exception-preview links present** — exceptions in the page are linked by at least one simulation via `exception_preview_refs`.
6. **Action-time policy truth preserved** — action-time policy snapshots carry `preserves_historical_truth: true`.

## Hard guardrail

- `raw_private_material_exposed` — a `RawPrivateMaterialExposed` defect in the
  upstream beta page forces `Withdrawn` for the entire packet immediately. No
  override is possible.

## Failure drills

Each `drill_*.json` fixture intentionally trips the validator:

- `drill_raw_private_material_withdrawn.json` raises `raw_private_material_exposed` (withdrawal reason).
- `drill_missing_change_class_preview.json` raises `required_change_class_missing` (preview narrowing).
- `drill_exception_preview_links_missing.json` raises `exception_preview_links_missing` (beta narrowing).
