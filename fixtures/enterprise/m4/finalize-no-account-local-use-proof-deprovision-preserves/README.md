# No-Account Local-Use Proof, Managed-Exit Truth, Deprovision-Preserves-Local-Work,
# and Org-Switch Semantics — Beta Fixtures

Reviewer-facing fixtures for the beta proof packet that proves local editing,
user-owned export paths, local history, local settings, and the account-free BYOK
lane survive every managed-exit event (sign-out, org-switch, seat loss, deprovision)
across all four required deployment profiles, and that org-scoped affordances are
removed or narrowed with explicit notice before the exit event completes.

The canonical record kind is `auth_deprovision_preserves_beta_page_record`. The
runtime module lives at
[`/crates/aureline-auth/src/finalize_no_account_local_use_proof_deprovision_preserves/mod.rs`](../../../../crates/aureline-auth/src/finalize_no_account_local_use_proof_deprovision_preserves/mod.rs).

## Files

These JSON files are generated by the headless example so they stay a literal
projection of the seeded `DeprovisionPreservesBetaPage` (no hand-written drift):

```sh
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- page > page.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- rows > rows.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- defects > defects.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- summary > summary.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- support-export > support_export.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- drill-silent-purge > drill_silent_purge_withdrawn.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- drill-blocking-exit > drill_blocking_exit_withdrawn.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- drill-missing-export-opportunity > drill_missing_export_opportunity.json
cargo run -q -p aureline-auth --example dump_deprovision_preserves_fixtures -- drill-affordance-without-notice > drill_affordance_without_notice.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta audit page (rows + defects + summary). |
| `rows.json` | All proof rows (one per exit-event × profile pair). |
| `defects.json` | Defect array (empty on the seeded page). |
| `summary.json` | Aggregate banner (row counts, profiles covered, exit events covered). |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `drill_silent_purge_withdrawn.json` | Drill: local editing silently purged on deprovision — row withdrawn. |
| `drill_blocking_exit_withdrawn.json` | Drill: sign-out reduces local editing to read-only — row withdrawn. |
| `drill_missing_export_opportunity.json` | Drill: org-switch row missing prior export opportunity — narrowed to beta. |
| `drill_affordance_without_notice.json` | Drill: collab session removed without notice on seat-loss — narrowed to beta. |

## Exit events covered

- `sign_out` — explicit user sign-out from managed identity
- `org_switch` — switch to a different org, tenant, or workspace identity
- `seat_loss` — managed seat revoked by admin or org policy
- `deprovision` — account deprovisioned (SCIM delete, admin-initiated, or signed-file revocation)
- `account_free_local_no_managed_exit` — account-free local lane (no exit events apply)

## Profiles covered

Each managed exit event is covered across all four required deployment profiles:
`connected`, `mirror_only`, `offline`, `enterprise_managed`.

## Failure drills

Each `drill_*.json` fixture intentionally trips the validator:

- `drill_silent_purge_withdrawn.json` raises `local_work_silently_purged`
  (withdrawal reason).
- `drill_blocking_exit_withdrawn.json` raises `managed_exit_blocks_local_core`
  (withdrawal reason).
- `drill_missing_export_opportunity.json` raises
  `export_path_unavailable_before_close` (beta narrowing).
- `drill_affordance_without_notice.json` raises
  `data_bearing_affordance_removed_without_notice` (beta narrowing).
