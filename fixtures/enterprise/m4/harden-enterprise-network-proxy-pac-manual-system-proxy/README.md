# Hardened Enterprise Network Proxy — PAC / Manual / System Proxy Fixtures

Reviewer-facing fixtures for the enterprise network proxy proof packet. The
packet proves that all six required proxy routes (system, PAC, manual,
policy-pinned, mirror-only, offline) are covered, each with explicit selector
reason, trust material declarations, TLS and client-cert state, and
local-only fallback documentation — and that no raw secret or private material
is exported at any boundary.

The canonical record kind is `policy_harden_enterprise_network_proxy_page_record`.
The runtime module lives at
[`/crates/aureline-policy/src/harden_enterprise_network_proxy_pac_manual_system_proxy/mod.rs`](../../../../crates/aureline-policy/src/harden_enterprise_network_proxy_pac_manual_system_proxy/mod.rs).

## Files

These JSON files are generated by the headless example so they stay a literal
projection of the seeded `HardenEnterpriseNetworkProxyPage` (no hand-written drift):

```sh
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- page > page.json
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- rows > rows.json
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- defects > defects.json
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- summary > summary.json
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- support-export > support_export.json
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- drill-raw-secret-withdrawn > drill_raw_secret_withdrawn.json
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- drill-missing-route-preview > drill_missing_route_preview.json
cargo run -q -p aureline-policy --example dump_harden_enterprise_network_proxy_fixtures -- drill-empty-selector-reason-beta > drill_empty_selector_reason_beta.json
```

| File | Purpose |
| --- | --- |

| `rows.json` | Row array only (one per proxy route). |

## Stability conditions verified

1. **Upstream network trust clean** — the embedded `NetworkTrustBetaPage` audits with zero defects.
2. **All six proxy routes covered** — `system`, `pac`, `manual`, `policy_pinned`, `mirror_only`, and `offline` each have a row.
3. **Raw secret material excluded** — every row carries `raw_secret_or_private_material_excluded: true`; no credential bodies, private keys, PAC script content, or CA certificates cross this boundary.
4. **Selector reasons declared** — every row carries a non-empty `selector_reason_token` from the closed vocabulary.
5. **Enterprise routes declare local fallbacks** — `policy_pinned`, `mirror_only`, and `offline` rows each carry a non-empty `local_only_fallback_route_token` and a human-readable `fallback_condition_label`.
6. **Bootstrap credentials declared** — every row carries at least one `BootstrapCredentialDeclaration` (minimum: `none_required`).
7. **TLS verification posture declared** — every row carries a non-empty `tls_verification_posture_token`.
8. **Local-core continuity explicit** — `mirror_only` and `offline` rows carry `local_core_continuity_explicit: true`.
9. **Managed attribution declared** — `policy_pinned` and `mirror_only` rows carry a non-empty `managed_attribution_ref`.

## Hard guardrails

One condition forces `Withdrawn` immediately and cannot be overridden:

- `raw_secret_or_private_material_exposed` — a row with
  `raw_secret_or_private_material_excluded: false` forces `Withdrawn` for
  the entire packet immediately.

## Failure drills

Each `drill_*.json` fixture intentionally trips the validator:

- `drill_raw_secret_withdrawn.json` raises `raw_secret_or_private_material_exposed` (withdrawal reason — system row has raw material).
- `drill_missing_route_preview.json` raises `missing_route_coverage` (preview narrowing — only system and PAC rows present).
- `drill_empty_selector_reason_beta.json` raises `empty_selector_reason` (beta narrowing — selector reason token cleared on first row).
