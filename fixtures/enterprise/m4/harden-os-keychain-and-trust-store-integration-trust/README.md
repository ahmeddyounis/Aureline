# Hardened OS Keychain and Trust-Store Integration — Fixtures

Reviewer-facing fixtures for the hardened OS keychain and trust-store integration
proof packet. The packet proves that all five required trust-store layers
(OS roots, custom CA bundle, pinned SSH host proof, client certificate, and
mirror trust root) are covered, each trust-store change event carries
attribution and a repair action, local-core continuity is explicitly documented
on every layer that requires it, and no raw trust material (certificate bodies,
private keys, fingerprints) is exported at any boundary.

The canonical record kind is `policy_harden_os_keychain_trust_store_page_record`.
The runtime module lives at
[`/crates/aureline-policy/src/harden_os_keychain_and_trust_store_integration_trust/mod.rs`](../../../../crates/aureline-policy/src/harden_os_keychain_and_trust_store_integration_trust/mod.rs).

## Files

These JSON files are generated by the headless example so they stay a literal
projection of the seeded `HardenOsKeychainTrustStorePage` (no hand-written drift):

```sh
cargo run -q -p aureline-policy --example dump_harden_os_keychain_trust_store_fixtures -- page > page.json
cargo run -q -p aureline-policy --example dump_harden_os_keychain_trust_store_fixtures -- rows > rows.json
cargo run -q -p aureline-policy --example dump_harden_os_keychain_trust_store_fixtures -- change-events > change_events.json
cargo run -q -p aureline-policy --example dump_harden_os_keychain_trust_store_fixtures -- defects > defects.json
cargo run -q -p aureline-policy --example dump_harden_os_keychain_trust_store_fixtures -- summary > summary.json
cargo run -q -p aureline-policy --example dump_harden_os_keychain_trust_store_fixtures -- support-export > support_export.json
cargo run -q -p aureline-policy --example dump_harden_os_keychain_trust_store_fixtures -- drill-raw-material-withdrawn > drill_raw_material_withdrawn.json
```

| --- | --- |
| `page.json` | Full page (5 rows + 5 change events + defects + summary). |
| `rows.json` | Layer-row array only (one per trust-store layer). |
| `change_events.json` | Change-event array (one per layer, with attribution and repair action). |

| `drill_missing_layer_preview.json` | Drill: only OS-roots and custom-CA-bundle rows present — narrowed to preview. |
| `drill_missing_local_continuity_beta.json` | Drill: local-continuity declaration absent on custom-CA-bundle row — narrowed to beta. |

## Stability conditions verified

1. **All five trust-store layers covered** — `os_roots`, `custom_ca_bundle`, `pinned_ssh_host_proof`, `client_certificate`, and `mirror_trust_root` each have a row.
2. **Raw trust material excluded** — every row and change event carries `raw_trust_material_excluded: true`; no certificate bodies, private keys, or raw fingerprints cross this boundary.
3. **Change events carry attribution** — every change event names a non-empty `attribution_token` from the closed vocabulary.
4. **Repair transaction IDs present** — every change event carries a non-empty `repair_transaction_id`.
5. **Route-blocking events name repair actions** — events with a blocking or pausing `session_impact_token` carry a typed `repair_action_token`.
6. **Local continuity explicit on required layers** — `custom_ca_bundle`, `pinned_ssh_host_proof`, and `mirror_trust_root` rows carry `local_continuity_explicit: true`.
7. **Managed-authority layers carry attribution refs** — `custom_ca_bundle`, `client_certificate`, and `mirror_trust_root` rows carry a non-empty `managed_attribution_ref`.
8. **Managed-authority events carry policy refs** — change events with `admin_policy_push` or `mirror_sync` attribution carry a non-empty `managed_policy_ref`.

## Hard guardrail

One condition forces `Withdrawn` immediately and cannot be overridden:

- `raw_trust_material_exposed` — a row or change event with
  `raw_trust_material_excluded: false` forces `Withdrawn` for the entire
  packet immediately.

## Failure drills

Each `drill_*.json` fixture intentionally trips the validator:

- `drill_raw_material_withdrawn.json` raises `raw_trust_material_exposed` (withdrawal reason — OS-roots row has raw material).
- `drill_missing_layer_preview.json` raises `missing_layer_coverage` (preview narrowing — only two layers present).
- `drill_missing_local_continuity_beta.json` raises `local_continuity_not_explicit` (beta narrowing — custom-CA-bundle row missing declaration).
