# Hardened Installation Topology — State-Root Audits, Silent Deployment, and Fleet Rollout Evidence Fixtures

Reviewer-facing fixtures for the managed-fleet installation topology proof packet. The
packet proves that every managed lane exposes organization/tenant identity, rollout-ring
state, updater owner, binary-root class, durable state-root topology, and policy source
in one inspectable admin/support view — and that silent deployment limits and full fleet
rollout evidence are declared for every managed lane.

The canonical record kind is `harden_install_topology_page_record`.
The runtime module lives at
[`/crates/aureline-install/src/harden_installation_topology_state_root_audits_silent_deployment/mod.rs`](../../../../crates/aureline-install/src/harden_installation_topology_state_root_audits_silent_deployment/mod.rs).

## Files

These JSON files are generated by the headless example so they stay a literal
projection of the seeded `HardenInstallTopologyPage` (no hand-written drift):

```sh
cargo run -q -p aureline-install --example dump_harden_install_topology_fixtures -- page > page.json
cargo run -q -p aureline-install --example dump_harden_install_topology_fixtures -- defects > defects.json
cargo run -q -p aureline-install --example dump_harden_install_topology_fixtures -- summary > summary.json
cargo run -q -p aureline-install --example dump_harden_install_topology_fixtures -- support-export > support_export.json
cargo run -q -p aureline-install --example dump_harden_install_topology_fixtures -- drill-no-managed-rows-preview > drill_no_managed_rows_preview.json
cargo run -q -p aureline-install --example dump_harden_install_topology_fixtures -- drill-admin-view-incomplete-withdrawn > drill_admin_view_incomplete_withdrawn.json
cargo run -q -p aureline-install --example dump_harden_install_topology_fixtures -- drill-silent-limits-not-declared-beta > drill_silent_limits_not_declared_beta.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full page (2 managed rows + 2 silent deployment rows + defects + summary). |
| `defects.json` | Defect array (empty on the seeded page — all conditions hold). |
| `summary.json` | Aggregate summary (row counts, rings covered, fleet evidence classes, qualification token). |
| `support_export.json` | Support-export wrapper with `raw_private_material_excluded: true` and full page projection. |
| `drill_no_managed_rows_preview.json` | Drill: empty managed_fleet_rows — packet narrowed to preview. |
| `drill_admin_view_incomplete_withdrawn.json` | Drill: admin_view_complete=false on first row — packet withdrawn. |
| `drill_silent_limits_not_declared_beta.json` | Drill: silent_deployment_limits_declared=false on first silent row — narrowed to beta. |

## Stability conditions verified

1. **Managed rows present** — at least one `ManagedFleetAuditRow` is declared.
2. **Admin view complete** — every row carries `admin_view_complete: true`; no managed lane hides topology from the admin support view.
3. **Tenant and policy refs non-empty** — every row carries a non-empty `tenant_ref` and `policy_source_ref`.
4. **State roots audited** — every managed row declares at least one `StateRootAuditEntry`.
5. **Full fleet evidence** — every managed row covers all seven required evidence classes: `ring_assignment`, `exact_build_inventory`, `managed_package_report`, `policy_root`, `rollback_target`, `verification_status`, and `support_export`.
6. **Silent deployment limits declared** — every `SilentDeploymentAuditRow` carries `limits_declared: true` and at least one entry in `disclosed_limits`.
7. **Return code families named** — every silent deployment row carries `return_code_families_named: true` and at least one `return_code_family_refs` entry.
8. **Raw private material excluded** — the support export always carries `raw_private_material_excluded: true`.

## Hard guardrails

One condition forces `Withdrawn` immediately and cannot be overridden:

- `admin_view_not_complete` — any managed row with `admin_view_complete: false` forces `Withdrawn`
  for the entire packet immediately, before any other check runs.

## Failure drills

Each `drill_*.json` fixture intentionally trips the validator:

- `drill_no_managed_rows_preview.json` raises `no_managed_fleet_rows_declared` (preview narrowing — managed_fleet_rows list is empty).
- `drill_admin_view_incomplete_withdrawn.json` raises `admin_view_not_complete` (withdrawal — first managed row has admin_view_complete=false).
- `drill_silent_limits_not_declared_beta.json` raises `silent_deployment_limits_not_declared` (beta narrowing — first silent row has limits_declared=false).
