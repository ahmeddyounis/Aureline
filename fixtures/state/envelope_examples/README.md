# Subscription envelope examples

Human-authored, reviewer-friendly single-envelope fixtures pinned to the
reactive-state prototype's scenario table. Each file is one
`SubscriptionEnvelope` rendered in the field order frozen in
`docs/adr/0005-subscription-envelope-and-invalidation-semantics.md` and
validated by `schemas/runtime/subscription_envelope.schema.json`.

These sit alongside the older decision-flow examples at
`fixtures/runtime/subscription_examples/` — the older set describes the
decision surface (ready / partial / stale / failed / imported / replayed);
this newer set pins the specific frames the reactive-state prototype
emits so reviewers can diff a frozen human expectation against the
auto-emitted artifacts under
`artifacts/state/invalidation_trace_examples/`. Snapshot-vs-delta parity
for one derived view is tracked separately in
`fixtures/state/snapshot_delta_parity_manifest.yaml`.

## Contract for readers

- Every file is a full envelope (no partial samples).
- Field order matches the ADR table; optional fields are present only
  when the scenario requires them.
- The `__fixture__` block at the top is reviewer metadata (name,
  scenario description, hooks exercised, ADR sections cited). The
  subscription envelope itself starts at `subscription_schema_version`.
- `subscription_id = 100` across files is the deterministic first
  allocation the prototype store makes; real runtimes allocate their
  own IDs.
- `producer_instance` values are synthetic (`synthetic-host/pid-*/boot-*/<producer_id>`)
  so the fixtures stay byte-stable across hosts.

## Regenerating (machine-emitted counterpart)

The counterpart artifacts under
`artifacts/state/invalidation_trace_examples/` are regenerated by
running:

```
./tools/reactive_proto.sh --emit-scenarios artifacts/state/invalidation_trace_examples
```

They are the byte-stable trace records the prototype produced; the
files in this directory are the human-authored, single-frame snapshots
a reviewer uses to sanity-check the schema and the vocabulary without
having to unpack a full trace.

## Coverage contract

Every scenario in `crates/aureline-reactive-state/src/harness.rs::SCENARIOS`
MUST have at least one fixture here that pins one of its featured
envelopes. Drift in either direction (orphan fixture without a scenario,
scenario without a fixture) is treated as a harness bug.
