# Settings repair, reset, import, and migration review fixtures

These fixtures are literal projections of the beta settings repair-plan
contract that lives in `crates/aureline-settings/src/repair_review/`.
Every file in this directory is regenerated by the
`aureline_settings_inspect` binary; never hand-edit them.

To regenerate (from the workspace root):

```
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-reset-value                   > fixtures/config/m3/settings_repair_and_reset/plan_reset_current_value.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-reset-section                 > fixtures/config/m3/settings_repair_and_reset/plan_reset_section_awaiting_checkpoint.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-reset-section-checkpointed    > fixtures/config/m3/settings_repair_and_reset/plan_reset_section_checkpointed.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-repair-drift                  > fixtures/config/m3/settings_repair_and_reset/plan_repair_drift.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-reapply-profile-fragment      > fixtures/config/m3/settings_repair_and_reset/plan_reapply_imported_profile_fragment.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-revert-migration-step         > fixtures/config/m3/settings_repair_and_reset/plan_revert_migration_step.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-adjacent-refused              > fixtures/config/m3/settings_repair_and_reset/plan_adjacent_refused.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-plan-policy-owned-refused          > fixtures/config/m3/settings_repair_and_reset/plan_policy_owned_refused.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-review-sheet                       > fixtures/config/m3/settings_repair_and_reset/review_sheet_reset_current_value.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- repair-support-export                     > fixtures/config/m3/settings_repair_and_reset/support_export.json
```

Replay coverage:

- `plan_reset_current_value.json` — single-value reset at workspace scope,
  scope-explicit, ready to apply.
- `plan_reset_section_awaiting_checkpoint.json` — multi-row section reset
  without a checkpoint; verdict is `awaiting_checkpoint`.
- `plan_reset_section_checkpointed.json` — same plan with the checkpoint
  preserved; verdict is `ready_to_apply`.
- `plan_repair_drift.json` — drift repair at user_global.
- `plan_reapply_imported_profile_fragment.json` — re-applies one fragment
  from an imported portable profile.
- `plan_revert_migration_step.json` — reverts a recorded migration step
  using a captured checkpoint.
- `plan_adjacent_refused.json` — hidden-reset guard refuses an unselected
  adjacent setting.
- `plan_policy_owned_refused.json` — refuses any user-initiated repair
  whose target scope is admin policy.
- `review_sheet_reset_current_value.json` — sheet projection wrapping the
  reset-value plan.
- `support_export.json` — set of plans (mixed accepted/declined/denied)
  bundled for support export.
