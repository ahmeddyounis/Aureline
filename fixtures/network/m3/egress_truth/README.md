# Network egress-truth beta fixtures

Reviewer-facing fixtures for the beta projection that labels every
network-capable beta surface (self-update, AI broker, connected providers,
extension marketplace, and docs / help) with the stable trio of
**egress class**, **origin scope**, and **locality** across the connected,
mirror-only, offline, and enterprise-managed profiles.

The canonical record kind is `network_network_badges_beta_page_record`. The
shared contract ref is `network:network_badges_beta:v1`. The module lives at
[`/crates/aureline-shell/src/network_badges/mod.rs`](../../../../crates/aureline-shell/src/network_badges/mod.rs)
and the headless inspector at
[`/crates/aureline-shell/src/bin/aureline_shell_network_badges.rs`](../../../../crates/aureline-shell/src/bin/aureline_shell_network_badges.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- support-rows > support_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- drill-locality-egress-mismatch > drill_locality_egress_mismatch.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- drill-hidden-public-cloud > drill_hidden_public_cloud.json
cargo run -q -p aureline-shell --bin aureline_shell_network_badges -- drill-missing-surface > drill_missing_surface.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page with rows, support rows, defects, and summary. |
| `rows.json` | One row per network-capable surface (update, AI, provider, extension, docs/help) with profile bindings. |
| `support_rows.json` | Export-safe support-row projection of every live row. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_locality_egress_mismatch.json` | Drill: a connected update binding declares `mirrored` locality while keeping `public_internet` egress, which fails `locality_inconsistent_with_egress`. |
| `drill_hidden_public_cloud.json` | Drill: a connected provider binding routes to `public_cloud` locality without naming the public route, which fails `hidden_public_cloud_routing`. |
| `drill_missing_surface.json` | Drill: the docs/help surface is removed, which fails `missing_surface_coverage`. |

## Protected states covered

- Every network-capable surface labels egress class, origin scope, and
  locality across all four profiles. Missing or wrong labels block beta
  promotion on the affected row.
- Public-cloud locality is never silent: bindings must name the public route
  in `route_label`, and bindings must declare `no_public_endpoint_fallback`
  so a mirror-only or offline profile cannot silently widen to the public
  internet.
- Locality vocabulary is consistent with the egress-class vocabulary used by
  transport governance: `local_only` ↔ {`local_only`, `deny_all`},
  `mirrored` ↔ `mirror_only`, `self_hosted` ↔ {`org_approved_external`,
  `target_local`}, `managed` ↔ {`public_internet`, `org_approved_external`},
  and `public_cloud` ↔ `public_internet`.
- Support exports report the same egress, origin, and locality tokens as the
  live page while keeping `raw_secret_or_private_material_excluded` set.
