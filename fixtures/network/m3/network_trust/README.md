# Network-trust beta fixtures

Reviewer-facing fixtures for the beta projection that proves enterprise
proxy, CA / trust-store, SSH host-proof, and client-certificate settings are
inspectable across connected, mirror-only, offline, and enterprise-managed
deployments — with their source and lock reason — and reused consistently
across runtime, extension, AI, provider, and update lanes.

The canonical record kind is `network_network_trust_beta_page_record`. The
schema lives at
[`/schemas/network/effective_network_config.schema.json`](../../../../schemas/network/effective_network_config.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/network_trust/mod.rs`](../../../../crates/aureline-auth/src/network_trust/mod.rs)
and the shell/headless consumer at
[`/crates/aureline-shell/src/network_trust_beta/mod.rs`](../../../../crates/aureline-shell/src/network_trust_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- support-rows > support_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- drill-unsigned-managed > drill_unsigned_managed.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- drill-public-fallback > drill_public_fallback.json
cargo run -q -p aureline-shell --bin aureline_shell_network_trust_beta -- drill-lock-source-mismatch > drill_lock_source_mismatch.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page with rows, support rows, defects, and summary. |
| `rows.json` | One row per facet (proxy, trust store, SSH host proof, client certificate) with profile bindings and consumer lanes. |
| `support_rows.json` | Export-safe support-row projection of every live row. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_unsigned_managed.json` | Drill: an enterprise-managed binding loses its signature pointer and must fail validation. |
| `drill_public_fallback.json` | Drill: a mirror-only proxy binding permits undeclared public fallback. |
| `drill_lock_source_mismatch.json` | Drill: a user-editable binding is marked as signed-managed-policy-locked. |

## Protected states covered

- Active proxy, CA / trust-store, SSH host-proof, and client-certificate
  settings are inspectable per profile with source and lock reason fields.
- Enterprise network settings declare a consistent fan-out across the
  runtime, extension, AI, provider, and update consumer lanes so the same
  effective value is reused without any lane re-deriving its own copy.
- Connected, mirror-only, offline, and enterprise-managed profile bindings
  refuse undeclared public endpoint fallback and exclude raw secret or
  private-key material from the record.
- Support exports report the same effective values, source tokens, and lock
  reasons as the live page while keeping `raw_secret_or_private_material_excluded`
  set.
