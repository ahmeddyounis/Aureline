# Component metrics ledger fixtures

Worked fixtures for the component metrics ledger and density-geometry contract
frozen in:

- [`/docs/design/component_metrics_contract.md`](../../../docs/design/component_metrics_contract.md)

and validated by:

- [`/schemas/design/component_metrics.schema.json`](../../../schemas/design/component_metrics.schema.json)

These cases exist to keep launch-critical surfaces aligned on:

- control sizes and row heights across density modes;
- minimum hit targets and splitter hit areas; and
- indentation/disclosure geometry and status-bar spacing budgets.

Cases are intentionally small and do not embed screenshots or assets; they bind
metric expectations to stable screenshot and accessibility review refs so drift
is reviewable before implementation hardens.
