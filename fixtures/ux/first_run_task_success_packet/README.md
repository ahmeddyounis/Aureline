# First-run task-success packet (beta)

These fixtures pin the bounded first-run task-success packet produced
by the shell projection in
[`crate::onboarding_metrics`](../../../crates/aureline-shell/src/onboarding_metrics/mod.rs).
The packet covers two claimed beta switching flows:

- the no-account first-run path that opens a local folder through the
  Start Center; and
- the imported-profile path that reviews a VS Code settings packet,
  mints a rollback checkpoint, and commits per-item outcomes.

Each flow is covered by four typed task-success states — `completion`,
`fallback`, `abandonment`, and `repair_required`. Imported-profile rows
declare `no_raw_sensitive_user_content = true` and the embedded
onboarding telemetry capture inherits the metadata-safe-default privacy
envelope, so dashboards, support exports, and partner scorecards never
see raw project content, file paths, prompt or terminal text, or
credentials.

## Files

| File | Inspector subcommand | What it pins |
|---|---|---|
| `packet.json` | `packet` | Full packet record. |
| `rows.json` | `rows` | Classified rows. |
| `summary.json` | `summary` | Per-flow state coverage summary. |
| `telemetry_capture.json` | `telemetry` | Onboarding task-success capture backing the rows. |
| `support_export.json` | `support-export` | Support-export wrapper with case ids. |
| `compact.txt` | `compact` | Headless compact summary lines. |

The markdown artifact lives at
[`artifacts/ux/m3/first_run_task_success_packet.md`](../../../artifacts/ux/m3/first_run_task_success_packet.md)
and is regenerated by the same headless inspector.

## Regenerating

```sh
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- packet \
  > fixtures/ux/first_run_task_success_packet/packet.json
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- rows \
  > fixtures/ux/first_run_task_success_packet/rows.json
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- summary \
  > fixtures/ux/first_run_task_success_packet/summary.json
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- telemetry \
  > fixtures/ux/first_run_task_success_packet/telemetry_capture.json
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- support-export \
  > fixtures/ux/first_run_task_success_packet/support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- compact \
  > fixtures/ux/first_run_task_success_packet/compact.txt
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- markdown \
  > artifacts/ux/m3/first_run_task_success_packet.md
```

The seeded projection is deterministic so the regenerated files are
bit-for-bit equal to what is checked in here.

## Validating

```sh
cargo run -q -p aureline-shell --bin aureline_shell_onboarding_metrics -- validate
```

The validator runs the same invariants as
`validate_first_run_task_success_packet` and exits non-zero with a
machine-readable list of detected violations when an invariant is
broken (missing flow/state cell, missing repair action, raw sensitive
content declared, stale state summary, …). The shared boundary schema
lives at
[`/schemas/ux/first_run_metrics.schema.json`](../../../schemas/ux/first_run_metrics.schema.json).
