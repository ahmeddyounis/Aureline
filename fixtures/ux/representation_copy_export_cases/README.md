# Representation copy/export parity cases

Worked fixtures for:

- [`docs/ux/copy_export_representation_parity.md`](../../../docs/ux/copy_export_representation_parity.md)
- [`schemas/ux/representation_copy_export.schema.json`](../../../schemas/ux/representation_copy_export.schema.json)

The corpus focuses on parity across preview, copy, and export surfaces:

- suspicious text that requires an escaped inspection path;
- Markdown/docs preview where raw vs rendered diverge;
- oversized/windowed output where scope truth must be explicit;
- binary or mixed bytes where only excerpts are transferable by default;
- generated content where origin/provenance and citation posture must be
  preserved; and
- support-export defaults (sanitized snapshot or metadata-only).
