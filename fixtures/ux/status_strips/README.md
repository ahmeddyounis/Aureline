# Status-strip fixtures

Worked examples for
[`docs/ux/status_strip_family_contract.md`](../../../docs/ux/status_strip_family_contract.md)
and
[`schemas/ux/status_strip.schema.json`](../../../schemas/ux/status_strip.schema.json).

The corpus covers the five strip families:

- `workspace_partial_hidden_scope.json`
- `environment_stale_target.json`
- `provider_blocked_policy.json`
- `graph_degraded_missing_lane.json`
- `framework_ready_exact_routes.json`

Together they exercise partial readiness, stale data, provider-backed
blocked state, degraded graph readiness, exact ready framework state,
top-level hidden-scope disclosure, command-backed details, and
export/screenshot-safe state cues.
