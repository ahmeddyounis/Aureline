# Action visibility cases

Worked fixtures for:

- [`docs/ux/action_discoverability_equivalence_contract.md`](../../../docs/ux/action_discoverability_equivalence_contract.md)
- [`schemas/ux/action_visibility_state.schema.json`](../../../schemas/ux/action_visibility_state.schema.json)

The corpus covers:

- dense list-row quick actions deferred to hover/focus plus overflow;
- tree-row inline actions with keyboard and screen-reader parity;
- table action cells that overflow without losing parity or explanations;
- inline toolbars with icon-only actions and tooltip + shortcut/palette routes;
- icon-only actions that are policy-limited and remain explainable; and
- overflow menus that preserve command identity, disabled reasons, and keyboard reachability.
