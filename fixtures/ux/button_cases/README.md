# Button cases

Worked fixtures for:

- [`docs/ux/button_family_contract.md`](../../../docs/ux/button_family_contract.md)
- [`schemas/ux/button_action.schema.json`](../../../schemas/ux/button_action.schema.json)

The corpus covers:

- `toolbar_icon_buttons.json` — icon-only toolbar actions with tooltip and
  keyboard parity plus minimum hit target.
- `dialog_action_row_destructive.json` — destructive dialog action row
  ordering with safe default focus.
- `split_button_routes.json` — split button default route, alternates,
  and branch auditability when side effects differ materially.
- `policy_locked_button.json` — policy-locked action rendered as locked,
  not disabled, with an inspect route.
- `pending_progress_button.json` — pending/progress action that preserves
  label width and meaning while work is in flight.

Fixtures carry opaque refs and review-safe labels only.
