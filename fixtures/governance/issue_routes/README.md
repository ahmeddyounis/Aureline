# Issue-routing worked examples

One worked example per seeded issue class from
[`/artifacts/governance/issue_routing.yaml`](../../../artifacts/governance/issue_routing.yaml).
Each file shows end-to-end routing: the default route, privacy and
disclosure posture, redaction posture, summary expectation, owning
forum, escalation chain, and — where relevant — the disclosure
transition a reviewer would cite if the posture changed.

Fixtures are machine-readable and deliberately short. They demonstrate
routing, not triage detail.
