# Managed-boundary beta fixtures

Reviewer-facing fixtures for the shell consumer that loads the published
boundary manifest and projects it into the org-switch, seat-quota,
grace-window, and offboarding flows.

The canonical machine source is the boundary manifest at
[`/artifacts/milestones/m3/boundary_manifest_beta.yaml`](../../../../artifacts/milestones/m3/boundary_manifest_beta.yaml),
governed by
[`/schemas/governance/boundary_manifest_beta.schema.json`](../../../../schemas/governance/boundary_manifest_beta.schema.json).
The shell module that projects the manifest into typed records lives at
[`/crates/aureline-shell/src/managed_boundary/mod.rs`](../../../../crates/aureline-shell/src/managed_boundary/mod.rs).
The reviewer-facing landing page is
[`/docs/ux/m3/managed_boundary_beta.md`](../../../../docs/ux/m3/managed_boundary_beta.md).

## Files

These JSON files are generated by the headless inspector and checked in
verbatim:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- org-switch-rows > org_switch_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- seat-quota-rows > seat_quota_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- grace-window-rows > grace_window_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- offboarding-rows > offboarding_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- summary > summary.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- render-summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_managed_boundary_beta -- support-export > support_export.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page: rows, per-flow blocks, defects, and summary. |
| `rows.json` | Typed boundary rows projected from the manifest. |
| `org_switch_rows.json` | Per-flow projection consumed by the org-switch surface. |
| `seat_quota_rows.json` | Per-flow projection consumed by the seat / quota surface. |
| `grace_window_rows.json` | Per-flow projection consumed by the grace-window surface. |
| `offboarding_rows.json` | Per-flow projection consumed by the offboarding surface. |
| `defects.json` | Defect array; empty on the published manifest. |
| `summary.json` | Aggregate summary with vocabulary coverage and defect roll-up. |
| `render_summary.json` | Shell rendering summary suitable for the surface and support-export clipboard. |
| `support_export.json` | Support-export wrapper carrying every per-flow projection and the invariant flags. |

## Protected states covered

- Org-switch, seat-quota, grace-window, and offboarding flows reuse the
  manifest vocabulary verbatim (no parallel labels are minted in product).
- Every row whose boundary class is `managed` or `paid_seat_bound`
  publishes an `absence_narrows_to` clause; the validator fails closed
  when that clause is missing.
- Every row preserves a `local_core_continuity` statement that names what
  still works locally when the managed surface is absent, denied, or
  offboarded.
- The support-export wrapper carries the typed defect roll-up, the
  no-public-endpoint invariant, the local-core-continuity invariant, and
  the absence-narrowing invariant alongside refs to the release doc and
  this surface's UX doc.
