# Policy-pack beta fixtures

Reviewer-facing fixtures for the beta projection that proves effective policy
packs, mirror and manual-import receipts, before/after diffs, and product-row
denial-explain traces share one inspectable vocabulary across connected,
mirror-only, offline, and enterprise-managed deployments.

The canonical record kind is `security_policy_pack_beta_page_record`. The
schema lives at
[`/schemas/security/policy_pack.schema.json`](../../../../schemas/security/policy_pack.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/policy_packs/mod.rs`](../../../../crates/aureline-auth/src/policy_packs/mod.rs)
and the shell/headless consumer at
[`/crates/aureline-shell/src/policy_pack_beta/mod.rs`](../../../../crates/aureline-shell/src/policy_pack_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- packs > packs.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- diffs > diffs.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- denial-traces > denial_traces.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- import-receipts > import_receipts.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- drill-signature-dropped > drill_signature_dropped.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- drill-denial-unresolvable > drill_denial_unresolvable.json
cargo run -q -p aureline-shell --bin aureline_shell_policy_pack_beta -- drill-public-fallback > drill_public_fallback.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page with packs, diffs, denial traces, import receipts, defects, and summary. |
| `packs.json` | Inspected packs across connected, mirror-only, offline, and enterprise-managed profiles. |
| `diffs.json` | Per-profile diffs comparing the baseline pack against the effective target. |
| `denial_traces.json` | Explain traces tying surface-row denials back to the originating pack and rule. |
| `import_receipts.json` | Mirror, manual-import, and air-gapped receipts that preserve signature, provenance, and explanation fields. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_signature_dropped.json` | Drill: a mirror import receipt drops its preserved signature blob and must fail validation. |
| `drill_denial_unresolvable.json` | Drill: a denial trace points at a rule that no longer exists. |
| `drill_public_fallback.json` | Drill: an effective pack permits undeclared public fallback. |

## Protected states covered

- Effective policy is inspectable: every pack exports source, signature state,
  apply state, provenance, and per-surface rules.
- Diff records show added, removed, effect-changed, scope-changed, and
  reason-changed rules so admins can review changes before applying.
- Mirror and manual-import receipts preserve signatures, provenance, and rule
  explanations verbatim from their upstream pack across signed mirror, manual
  signed-file import, and air-gapped signed transport.
- Denial traces tie product-surface denials back to the same pack id, rule id,
  surface family token, and reason token they originated from.
- Connected, mirror-only, offline, and enterprise-managed profile rows all
  refuse undeclared public endpoint fallback and exclude raw private material.
