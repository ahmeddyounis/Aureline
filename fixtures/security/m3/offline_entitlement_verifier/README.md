# Offline policy-bundle and entitlement verifier beta fixtures

Reviewer-facing fixtures for the beta projection that proves the signed
policy-bundle and offline entitlement verifier produces one inspectable
record per bundle, per profile, per kind across connected, mirror-only,
offline, and enterprise-managed deployments, and that failure modes
downgrade managed authority while preserving local editing.

The canonical record kind is
`security_offline_entitlement_verifier_beta_page_record`. The schema
lives at
[`/schemas/security/offline_entitlement.schema.json`](../../../../schemas/security/offline_entitlement.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/offline_entitlements/mod.rs`](../../../../crates/aureline-auth/src/offline_entitlements/mod.rs)
and the shell/headless consumer at
[`/crates/aureline-shell/src/offline_entitlement_beta/mod.rs`](../../../../crates/aureline-shell/src/offline_entitlement_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- drill-expired-bundle-accepted > drill_expired_bundle_accepted.json
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- drill-untrusted-signer-accepted > drill_untrusted_signer_accepted.json
cargo run -q -p aureline-shell --bin aureline_shell_offline_entitlement_beta -- drill-local-editing-blocked > drill_local_editing_blocked.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page with rows, defects, and summary. |
| `rows.json` | Verifier rows across connected, mirror-only, offline, and enterprise-managed profiles for both `policy_bundle` and `entitlement_snapshot` kinds. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up and per-row support projections. |
| `render_summary.json` | Shell rendering summary. |
| `drill_expired_bundle_accepted.json` | Drill: an expired bundle row claims full managed authority and must fail validation. |
| `drill_untrusted_signer_accepted.json` | Drill: an untrusted-signer row claims full managed authority and must fail validation. |
| `drill_local_editing_blocked.json` | Drill: a failed-verify row blocks local editing and must fail validation. |

## Protected states covered

- Verified bundles produce `full_authority_active` managed impact under
  connected, mirror-only, offline, and enterprise-managed profiles for
  both policy and entitlement bundle kinds.
- Expired offline entitlement snapshots narrow managed authority to
  inspect-only while preserving local editing with an advisory.
- Missing-signature enterprise entitlement snapshots pause managed
  actions with a visible recovery cue while preserving local editing.
- Untrusted-signer policy bundles block managed authority pending admin
  quorum review while preserving local editing.
- Unsigned local-advisory bundles are inspectable but carry no managed
  authority and live on the `local_advisory_no_root` trust anchor.
- All rows refuse undeclared public endpoint fallback and exclude raw
  private/secret material from the record itself.
