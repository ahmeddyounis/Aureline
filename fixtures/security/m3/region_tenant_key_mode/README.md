# Region / tenant / key-mode beta fixtures

Reviewer-facing fixtures for the beta projection that turns region pinning,
tenant boundary, and key-mode truth into one inspectable record kind across
connected, mirror-only, offline, and enterprise-managed beta profiles. The
fixtures pair the page with the region, tenant, and key-mode drill packets in
[`/artifacts/security/m3/region_tenant_drills/`](../../../../artifacts/security/m3/region_tenant_drills/).

The canonical record kind is
`security_region_tenant_key_mode_beta_page_record`. The schema lives at
[`/schemas/security/region_tenant_key_mode_beta.schema.json`](../../../../schemas/security/region_tenant_key_mode_beta.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/region_and_tenant/mod.rs`](../../../../crates/aureline-auth/src/region_and_tenant/mod.rs)
and the shell consumer at
[`/crates/aureline-shell/src/region_tenant_key_mode_beta/mod.rs`](../../../../crates/aureline-shell/src/region_tenant_key_mode_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- region-rows > region_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- tenant-rows > tenant_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- key-mode-rows > key_mode_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- drill-packets > drill_packets.json
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_region_tenant_key_mode_beta -- summary > render_summary.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page with region, tenant, key-mode rows, drill packets, defects, and summary. |
| `region_rows.json` | Region disclosure rows across all four profiles. |
| `tenant_rows.json` | Tenant boundary rows across all four profiles. |
| `key_mode_rows.json` | Key-mode rows across all four profiles. |
| `drill_packets.json` | Region, tenant, and key-mode drill packets covering failure and failover scenarios. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up and invariant flags. |
| `render_summary.json` | Shell rendering summary. |

## Protected states covered

- Claimed managed / enterprise rows disclose region, tenant, and key-mode
  truth in product and in the exported packet.
- Mismatch or degraded states (`pinning_lost`, `drifted_from_claim`,
  `binding_lost`, `custody_degraded`, `unresolved`) narrow only the affected
  managed lane.
- At least one drill packet exists for each of the three required axes:
  region, tenant, and key-mode.
- Every row and drill refuses undeclared public-endpoint fallback and excludes
  raw private material.
- Local editing is preserved across every row and drill.

Enterprise docs and support playbooks reference the same exported defect
vocabulary as the auditing surface — see
[`/docs/security/m3/region_tenant_key_mode_beta.md`](../../../../docs/security/m3/region_tenant_key_mode_beta.md).
