# Admin-audit export beta fixtures

Reviewer-facing fixtures for the beta projection that turns SCIM and
signed-file provisioning hooks, policy-bundle history, and entitlement changes
into one inspectable admin-audit export across connected, mirror-only,
offline, and enterprise-managed beta profiles.

The canonical record kind is
`security_admin_audit_export_beta_page_record`. The schema lives at
[`/schemas/security/admin_audit_export_beta.schema.json`](../../../../schemas/security/admin_audit_export_beta.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/provisioning/mod.rs`](../../../../crates/aureline-auth/src/provisioning/mod.rs)
and the shell/headless consumer at
[`/crates/aureline-shell/src/admin_audit_export_beta/mod.rs`](../../../../crates/aureline-shell/src/admin_audit_export_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- provisioning-events > provisioning_events.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- policy-bundle-history > policy_bundle_history.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- entitlement-changes > entitlement_changes.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- drill-managed-source-missing-signature > drill_managed_source_missing_signature.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- drill-history-missing-predecessor > drill_history_missing_predecessor.json
cargo run -q -p aureline-shell --bin aureline_shell_admin_audit_export_beta -- drill-public-fallback > drill_public_fallback.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page with provisioning events, policy-bundle history, entitlement changes, defects, and summary. |
| `provisioning_events.json` | SCIM and signed-file provisioning hook events across all four profiles. |
| `policy_bundle_history.json` | Applied / replaced / staged / rolled-back / revoked transitions per profile. |
| `entitlement_changes.json` | Granted, revoked, scope-narrowed, seat-removed, and expiry-shortened changes with before / after authority tokens. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_managed_source_missing_signature.json` | Drill: a SCIM event drops its signature blob ref and must fail validation. |
| `drill_history_missing_predecessor.json` | Drill: a `replaced_by_successor` history event loses its predecessor pack id. |
| `drill_public_fallback.json` | Drill: a provisioning hook permits undeclared public fallback. |

## Protected states covered

- Provisioning hooks and signed-file alternatives are explicit about source,
  freshness, and lifecycle state. Stale, expired, and missing postures fail
  closed and never widen managed authority silently.
- Admins can export audit views that show policy-bundle history, provisioning
  events, and entitlement changes from one record kind. Mirror, manual, and
  air-gapped lanes preserve signer, signed-at, signature blob ref, and origin
  ref verbatim.
- Connected, mirror-only, offline, and enterprise-managed profile rows all
  refuse undeclared public endpoint fallback and exclude raw private material.
- Enterprise docs and support playbooks reference the same exported defect
  vocabulary as the auditing surface — see
  [`/docs/security/m3/provisioning_and_audit_beta.md`](../../../../docs/security/m3/provisioning_and_audit_beta.md).
