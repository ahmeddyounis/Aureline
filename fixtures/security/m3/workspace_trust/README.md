# Workspace-trust beta fixtures

Reviewer-facing fixtures for the beta projection that proves every claimed
workspace-trust lane has restricted-mode behavior before trust, explicit
trust-elevation and trust-loss vocabulary, profile coverage for connected,
mirror-only, offline, and enterprise-managed deployments, and export-safe
support rows.

The canonical record kind is
`security_workspace_trust_beta_page_record`. The schema lives at
[`/schemas/security/workspace_trust_beta.schema.json`](../../../../schemas/security/workspace_trust_beta.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/workspace_trust/mod.rs`](../../../../crates/aureline-auth/src/workspace_trust/mod.rs)
and the shell/headless consumer at
[`/crates/aureline-shell/src/workspace_trust_beta/mod.rs`](../../../../crates/aureline-shell/src/workspace_trust_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- support-rows > support_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- drill-trust-bypass > drill_trust_bypass.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- drill-support-drift > drill_support_drift.json
cargo run -q -p aureline-shell --bin aureline_shell_workspace_trust_beta -- drill-public-fallback > drill_public_fallback.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta audit page with rows, support rows, defects, and summary. |
| `rows.json` | Live audited rows grouped by open, run, debug, extension, AI, provider, review, support, and admin lanes. |
| `support_rows.json` | Export-safe support rows aligned 1:1 with live rows. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_trust_bypass.json` | Drill: `tasks_run` is allowed before trust and must fail validation. |
| `drill_support_drift.json` | Drill: support row authority drifts from the live row. |
| `drill_public_fallback.json` | Drill: connected-provider tool call permits undeclared public fallback. |

## Protected states covered

- Restricted mode keeps open, edit, search, local Git inspection, admin-policy
  read, and support export available.
- Run-capable and mutation-capable rows such as tasks, terminals, debug,
  extension activation, AI apply, provider tool calls, remote attach, and MCP
  launch are blocked or review-gated before trust.
- Trust elevation emits `workspace_trust_granted`; trust loss emits
  `workspace_trust_revoked` and keeps shell, runtime, extension, and support
  propagation booleans true on every row.
- Connected, mirror-only, offline, and enterprise-managed profile rows all
  refuse undeclared public endpoint fallback and exclude raw private material.
