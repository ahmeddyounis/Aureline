# Secret-broker beta fixtures

Reviewer-facing fixtures for the beta projection that delivers handle-only
secret-broker rows paired with a consumer-identity audit stream across
connected, mirror-only, offline, and enterprise-managed beta profiles.

The canonical record kind is `security_secret_broker_beta_page_record`. The
schema lives at
[`/schemas/security/secret_handle.schema.json`](../../../../schemas/security/secret_handle.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/secret_broker/mod.rs`](../../../../crates/aureline-auth/src/secret_broker/mod.rs)
and the shell/headless consumer at
[`/crates/aureline-shell/src/secret_broker_beta/mod.rs`](../../../../crates/aureline-shell/src/secret_broker_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- handle-rows > handle_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- consumer-audit > consumer_audit.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- drill-raw-secret-material > drill_raw_secret_material.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- drill-managed-authority-missing-signature > drill_managed_authority_missing_signature.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- drill-consumer-audit-missing > drill_consumer_audit_missing.json
cargo run -q -p aureline-shell --bin aureline_shell_secret_broker_beta -- drill-denied-audit-missing-reason > drill_denied_audit_missing_reason.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta page: handle rows, consumer audit, defects, summary. |
| `handle_rows.json` | Handle-only projection rows across all four profiles. |
| `consumer_audit.json` | Consumer-identity audit events with granted and typed-denial outcomes. |
| `defects.json` | Defect array; empty on the seeded page. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_raw_secret_material.json` | Drill: a row claims raw secret material is present. |
| `drill_managed_authority_missing_signature.json` | Drill: a managed-authority vault adapter loses its verified signature posture. |
| `drill_consumer_audit_missing.json` | Drill: a handle row is left without any consumer audit events. |
| `drill_denied_audit_missing_reason.json` | Drill: a denied audit event drops its denial note. |

## Protected states covered

- Beta rows present a handle, delegated credential, or visibly degraded
  session-only reference; raw secret bytes and runtime handle ids never
  travel.
- Every claimed row appears in the consumer-identity audit stream with at
  least one event; denial events name a typed reason and carry a denial
  note.
- Managed-authority vault adapters require a verified signature posture and
  a preserved signature blob ref. Loss of either fails closed.
- All four beta profiles (connected, mirror-only, offline, enterprise
  managed) appear on the page. Profile drift is surfaced as a typed
  defect.
- The redaction-safe support export preserves vault adapter, signature
  posture, projection mode, consumer lineage, target/scope refs, and typed
  audit outcomes — see
  [`/docs/security/m3/secret_broker_beta.md`](../../../../docs/security/m3/secret_broker_beta.md).
