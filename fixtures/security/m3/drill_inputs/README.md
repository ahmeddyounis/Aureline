# Enterprise drill baseline fixtures

Reviewer-facing fixtures for the first backup-restore, failover, and
key-rotation drill baseline that pairs every claimed managed / enterprise row
family with three drill kinds across connected, mirror-only, offline, and
enterprise-managed beta profiles. The fixtures pair the page with the drill
artifacts in
[`/artifacts/security/m3/backup_restore_failover_drills/`](../../../../artifacts/security/m3/backup_restore_failover_drills/).

The canonical record kind is
`security_enterprise_drill_baseline_page_record`. The schema lives at
[`/schemas/security/enterprise_drill_baseline.schema.json`](../../../../schemas/security/enterprise_drill_baseline.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/enterprise_drill_baseline/mod.rs`](../../../../crates/aureline-auth/src/enterprise_drill_baseline/mod.rs)
and the shell consumer at
[`/crates/aureline-shell/src/enterprise_drill_baseline/mod.rs`](../../../../crates/aureline-shell/src/enterprise_drill_baseline/mod.rs).

## Files

These JSON files are generated by the headless inspector:

```sh
cargo run -q -p aureline-shell --bin aureline_shell_enterprise_drill_baseline -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_enterprise_drill_baseline -- drill-packets > drill_packets.json
cargo run -q -p aureline-shell --bin aureline_shell_enterprise_drill_baseline -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_enterprise_drill_baseline -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_enterprise_drill_baseline -- summary > render_summary.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full drill baseline page with drill packets, defects, and summary. |
| `drill_packets.json` | All seeded drill packets (3 row families × 3 drill kinds = 9 packets). |
| `defects.json` | Defect array; empty on the seeded baseline. |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up and invariant flags. |
| `render_summary.json` | Shell rendering summary. |

## Protected states covered

- Each claimed managed / enterprise row family carries at least one current
  drill packet for backup_restore, failover, and key_rotation.
- Drill packets feed claim downgrades: stale_beyond_window and missing
  evidence states declare a non-no_impact claim impact; fresh evidence never
  asserts a downgrade.
- A drill never widens authority on a sibling lane, never falls back to a
  public endpoint, never exposes raw private material, and never blocks local
  editing.
- A drill outcome must match its drill kind (backup_restore drills cannot
  record failover outcomes and vice versa).

Enterprise docs and support playbooks reference the same exported defect
vocabulary as the auditing surface — see
[`/docs/security/m3/enterprise_drill_baseline.md`](../../../../docs/security/m3/enterprise_drill_baseline.md).
