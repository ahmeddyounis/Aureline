# Mirrored Docs-Pack Recall Fixtures

These fixtures are recall-packet materialization inputs (`input`) paired with the
`promotion_state` and `expected_finding_kinds` the validator must produce. They
are regenerated by:

```sh
cargo run -q -p aureline-docs --bin aureline_docs_pack_recall -- fixture <case_name>
```

The canonical clean-Stable recall lives in
[`artifacts/docs/m5/implement_mirrored_docs_pack_recall_source_or_version_or_freshness_chips_and_stale_example_findings/support_export.json`](../../../../artifacts/docs/m5/implement_mirrored_docs_pack_recall_source_or_version_or_freshness_chips_and_stale_example_findings/support_export.json);
these fixtures exercise the downgrade and blocking paths around it.

## mirror_offline_recall_narrowed.json

The pinned, signed mirror is offline. Its row falls back to the last verified
offline snapshot with `refresh_pending` freshness and `medium` confidence, and
its nearby-version finding becomes `narrowing`. The recall stays a valid,
attributable packet but narrows to `narrowed_below_stable` instead of hiding the
result — the downgrade narrows the claim, it does not hide the lane.

## live_mirror_over_authoritative_blocks_stable.json

A `mirrored_official_docs` row claims `authoritative_live` freshness while its
pack is neither pinned nor signature-verified, making the mirror look more
authoritative than it is. The validator blocks promotion with
`live_mirror_looks_more_authoritative`.

## stale_state_collapsed_blocks_stable.json

A row whose version drifted incompatibly is presented as a high-confidence,
`authoritative_live` match, collapsing the version-match chip's truth. The
validator blocks promotion with `version_truth_collapsed`.
