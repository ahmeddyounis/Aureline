# Light Remote Edit Surface Fixtures

These fixtures are light-remote-edit packet materialization inputs (`input`)
paired with the `promotion_state` and `expected_finding_kinds` the validator must
produce. They are regenerated by:

```sh
cargo run -q -p aureline-docs --bin aureline_docs_light_remote_edit_surfaces -- fixture <case_name>
```

The canonical clean-Stable light-remote-edit set lives in
[`artifacts/docs/m5/add_browser_lite_light_remote_edit_surfaces_with_narrow_scope_stale_state_honesty_and_no_hidden_authority_expa/support_export.json`](../../../../artifacts/docs/m5/add_browser_lite_light_remote_edit_surfaces_with_narrow_scope_stale_state_honesty_and_no_hidden_authority_expa/support_export.json);
these fixtures exercise the downgrade and blocking paths around it.

## authority_narrowed_rerun_narrowed.json

The remote single-file edit's authority was narrowed to read-only after a policy
change, so the set records a narrowing `authority_narrowed` degradation. The
surfaces stay valid and attributable but narrow to `narrowed_below_stable`
instead of hiding them — the downgrade narrows the claim, it does not hide the
surface.

## stale_base_undisclosed_blocks_stable.json

A single-file remote edit is prepared against a known-stale snapshot but does not
disclose it. Stale-state honesty is mandatory, so the validator blocks promotion
with `stale_state_not_disclosed`.

## authority_expansion_blocks_stable.json

A single-file remote edit exercises single-file-write authority while only
single-field-write was granted. A light remote edit may never expand its
authority beyond the grant, so the validator blocks promotion with
`authority_expansion_detected`.
