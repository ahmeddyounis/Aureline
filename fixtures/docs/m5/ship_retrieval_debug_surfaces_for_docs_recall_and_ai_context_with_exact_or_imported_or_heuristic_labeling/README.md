# Retrieval-Debug Surface Fixtures

These fixtures are retrieval-debug packet materialization inputs (`input`) paired
with the `promotion_state` and `expected_finding_kinds` the validator must
produce. They are regenerated by:

```sh
cargo run -q -p aureline-docs --bin aureline_docs_retrieval_debug_surfaces -- fixture <case_name>
```

The canonical clean-Stable retrieval-debug set lives in
[`artifacts/docs/m5/ship_retrieval_debug_surfaces_for_docs_recall_and_ai_context_with_exact_or_imported_or_heuristic_labeling/support_export.json`](../../../../artifacts/docs/m5/ship_retrieval_debug_surfaces_for_docs_recall_and_ai_context_with_exact_or_imported_or_heuristic_labeling/support_export.json);
these fixtures exercise the downgrade and blocking paths around it.

## index_stale_narrowed.json

The recall index is stale relative to the working tree, so the set records a
narrowing `index_stale` degradation. The entries stay valid and attributable but
narrow to `narrowed_below_stable` instead of hiding the results — the downgrade
narrows the claim, it does not hide the lane.

## uncited_imported_entry_blocks_stable.json

An imported semantic-recall entry drops its citation. Imported results must stay
cited, so the validator blocks promotion with `entry_not_cited`.

## heuristic_entry_high_confidence_blocks_stable.json

A heuristic AI-context entry is presented as a `high`-confidence result. A
heuristic match may never read as high confidence, so the validator blocks
promotion with `heuristic_label_looks_authoritative`.
