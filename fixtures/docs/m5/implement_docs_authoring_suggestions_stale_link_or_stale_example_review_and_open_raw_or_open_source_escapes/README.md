# Docs Authoring-and-Review Fixtures

These fixtures are docs-authoring-review packet materialization inputs (`input`)
paired with the `promotion_state` and `expected_finding_kinds` the validator must
produce. They are regenerated by:

```sh
cargo run -q -p aureline-docs --bin aureline_docs_authoring_review -- fixture <case_name>
```

The canonical clean-Stable review set lives in
[`artifacts/docs/m5/implement_docs_authoring_suggestions_stale_link_or_stale_example_review_and_open_raw_or_open_source_escapes/support_export.json`](../../../../artifacts/docs/m5/implement_docs_authoring_suggestions_stale_link_or_stale_example_review_and_open_raw_or_open_source_escapes/support_export.json);
these fixtures exercise the downgrade and blocking paths around it.

## stale_example_drift_narrows.json

A stale-example review finds the retry/backoff example drifted from the documented
signature. The verdict carries a narrowing severity and the example is shown as
stale, so the set narrows to `narrowed_below_stable` instead of hiding it — the
downgrade narrows the claim, it does not hide the item.

## broken_link_blocks_stable.json

A stale-link review finds the runbook link broken but the item still claims
live-authoritative freshness. Stale-link review truth is mandatory, so the
validator blocks promotion with `stale_verdict_freshness_mismatch` — a broken link
can never read as current.

## unverified_suggestion_apply_blocks_stable.json

An unverified live-mirror authoring suggestion offers a one-click apply.
Authoring-suggestion apply-posture truth is mandatory, so the validator blocks
promotion with `unverified_suggestion_apply_offered` — an unverified suggestion may
surface a preview but never a one-click apply.
