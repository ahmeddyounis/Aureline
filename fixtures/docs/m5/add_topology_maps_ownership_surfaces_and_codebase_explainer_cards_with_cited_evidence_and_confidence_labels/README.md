# Topology, Ownership, and Codebase Explainer Card Fixtures

These fixtures are card-packet materialization inputs (`input`) paired with the
`promotion_state` and `expected_finding_kinds` the validator must produce. They
are regenerated by:

```sh
cargo run -q -p aureline-docs --bin aureline_docs_codebase_understanding_cards -- fixture <case_name>
```

The canonical clean-Stable card set lives in
[`artifacts/docs/m5/add_topology_maps_ownership_surfaces_and_codebase_explainer_cards_with_cited_evidence_and_confidence_labels/support_export.json`](../../../../artifacts/docs/m5/add_topology_maps_ownership_surfaces_and_codebase_explainer_cards_with_cited_evidence_and_confidence_labels/support_export.json);
these fixtures exercise the downgrade and blocking paths around it.

## graph_index_stale_narrowed.json

The workspace graph index is stale relative to the working tree, so the card set
records a narrowing `graph_index_stale` degradation. The cards stay valid and
attributable but narrow to `narrowed_below_stable` instead of hiding the topology
— the downgrade narrows the claim, it does not hide the lane.

## uncited_topology_card_blocks_stable.json

A derived topology card drops its citation. A topology card must stay cited, so
the validator blocks promotion with `card_not_cited`.

## inferred_explainer_over_authoritative_blocks_stable.json

An inferred codebase explainer is presented as a `high`-confidence card.
Inference must preserve confidence, so the validator blocks promotion with
`inferred_card_looks_authoritative`.
