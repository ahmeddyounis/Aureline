# Passkey-capable step-up, reauth, and recovery lanes beta fixtures

Reviewer-facing fixtures for the beta projection that proves every claimed
beta row participating in a passkey-bound lane (step-up before a risky action,
reauth after session expiry, or recovery after device loss) names its lane,
lifecycle state, client scope, outcome, and a typed fallback when the passkey
cannot be satisfied; preserves the originating target / action identity across
reauth and recovery; and never widens granted authority beyond the requested
scope.

The canonical record kind is `auth_passkey_step_up_beta_page_record`. The
schema lives at
[`/schemas/auth/passkey_step_up_beta.schema.json`](../../../../schemas/auth/passkey_step_up_beta.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/passkey/mod.rs`](../../../../crates/aureline-auth/src/passkey/mod.rs)
and the live shell consumer at
[`/crates/aureline-shell/src/passkey_step_up_beta/mod.rs`](../../../../crates/aureline-shell/src/passkey_step_up_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector so they stay a
literal projection of the seeded `PasskeyStepUpBetaPage` (no hand-written
drift):

```sh
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- support-rows > support_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- drill-recovery-rerouted > drill_recovery_rerouted.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- drill-reauth-widened > drill_reauth_widened.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- drill-reauth-without-fallback > drill_reauth_without_fallback.json
cargo run -q -p aureline-shell --bin aureline_shell_passkey_step_up_beta -- drill-step-up-widens-authority > drill_step_up_widens_authority.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta audit page (rows + support rows + defects + summary). |
| `rows.json` | Live audited rows. |
| `support_rows.json` | Export-safe support rows. |
| `defects.json` | Defect array (empty on the seeded page). |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_recovery_rerouted.json` | Drill: recovery lane reroutes the original target / action identity. |
| `drill_reauth_widened.json` | Drill: reauth lane widens the original target / action identity. |
| `drill_reauth_without_fallback.json` | Drill: reauth lane drops the typed fallback path while the lane is unsatisfied. |
| `drill_step_up_widens_authority.json` | Drill: step-up row grants `tenant_admin_scope` despite requesting `step_up_scope`. |

## Lanes covered

- `step_up_lane` satisfied by an `active_on_this_device` passkey before a
  risky write action; granted scope equals the requested step-up scope.
- `reauth_lane` on an `unavailable_this_platform` row that names
  `resume_after_step_up_in_system_browser` as the typed fallback; the
  originating managed-sign-in target is preserved exactly.
- `recovery_lane` where admin policy denies passkey rebind on a `revoked`
  lifecycle state; the row names `contact_admin_for_recovery` as the fallback,
  preserves the original recovery target, and downscopes from `read_write` to
  `read_only` (preserved + downscoped — no widening).
- `not_applicable_account_free_local` row that quotes the local sentinel
  tokens across lane, lifecycle, scope, outcome, fallback, and preservation.

## Failure drills

Each `drill_*.json` fixture intentionally trips the validator:

- `drill_recovery_rerouted.json` raises `reauth_or_recovery_rerouted`.
- `drill_reauth_widened.json` raises `reauth_or_recovery_widened`.
- `drill_reauth_without_fallback.json` raises either
  `outcome_unsatisfied_without_fallback` or
  `lifecycle_state_requires_fallback`.
- `drill_step_up_widens_authority.json` raises
  `granted_authority_widens_requested`.

Each drill is replayed end-to-end by the integration test at
[`crates/aureline-shell/tests/passkey_step_up_beta_fixtures.rs`](../../../../crates/aureline-shell/tests/passkey_step_up_beta_fixtures.rs).
