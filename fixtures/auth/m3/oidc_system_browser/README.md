# OIDC system-browser sign-in, recovery, and session-continuity beta fixtures

Reviewer-facing fixtures for the beta projection that proves every claimed
enterprise OIDC row discloses its issuer source, tenant + workspace binding,
and return-path; degrades truthfully on identity outage or denial; and
preserves local editing through sign-out and refresh.

The canonical record kind is `auth_oidc_system_browser_beta_page_record`. The
schema lives at
[`/schemas/auth/oidc_system_browser_beta.schema.json`](../../../../schemas/auth/oidc_system_browser_beta.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/oidc/mod.rs`](../../../../crates/aureline-auth/src/oidc/mod.rs)
and the live shell consumer at
[`/crates/aureline-shell/src/oidc_system_browser_beta/mod.rs`](../../../../crates/aureline-shell/src/oidc_system_browser_beta/mod.rs).

## Files

These JSON files are generated by the headless inspector so they stay a
literal projection of the seeded `OidcSystemBrowserBetaPage` (no hand-written
drift):

```sh
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- support-rows > support_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- drill-public-issuer-fallback > drill_public_issuer_fallback.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- drill-sign-out-loses-local-editing > drill_sign_out_loses_local_editing.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- drill-outage-widens-authority > drill_outage_widens_authority.json
cargo run -q -p aureline-shell --bin aureline_shell_oidc_system_browser_beta -- drill-outage-missing-class > drill_outage_missing_class.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta audit page (rows + support rows + defects + summary). |
| `rows.json` | Live audited rows. |
| `support_rows.json` | Export-safe support rows. |
| `defects.json` | Defect array (empty on the seeded page). |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_public_issuer_fallback.json` | Drill: row silently falls back to a public issuer endpoint. |
| `drill_sign_out_loses_local_editing.json` | Drill: signed-out row narrows local editing. |
| `drill_outage_widens_authority.json` | Drill: issuer outage row claims `tenant_admin_scope` despite requested `read_write_scope`. |
| `drill_outage_missing_class.json` | Drill: issuer-outage row still quotes `no_outage` as the outage class. |

## Protected states covered

- `signed_in_active` on a managed enterprise OIDC issuer with a verified JWKS
  source, `loopback_port_pinned` origin validation, and
  `must_match_bound_workspace_and_tenant`.
- `refresh_expired_managed_blocked` keeps local editing intact while narrowing
  managed actions, with `retry_refresh_in_system_browser` as the recovery
  action.
- `signed_out_local_intact` preserves local editing while managed actions
  pause; `resume_in_system_browser` is the recovery action.
- `identity_outage_managed_blocked` degrades to local-only without falling
  back to a public issuer; `continue_local_without_sign_in` is the recovery
  action.
- `account_free_local_no_auth_required` quotes the account-free local issuer
  source, no tenant binding, and grants no remote scope.

## Failure drills

Each `drill_*.json` fixture intentionally trips the validator:

- `drill_public_issuer_fallback.json` raises `public_issuer_fallback_used`.
- `drill_sign_out_loses_local_editing.json` raises
  `sign_out_or_outage_loses_local_editing`.
- `drill_outage_widens_authority.json` raises `return_widens_authority_scope`.
- `drill_outage_missing_class.json` raises `identity_outage_missing_class`.

Each drill is replayed end-to-end by the integration test at
[`crates/aureline-shell/tests/oidc_system_browser_beta_fixtures.rs`](../../../../crates/aureline-shell/tests/oidc_system_browser_beta_fixtures.rs).
