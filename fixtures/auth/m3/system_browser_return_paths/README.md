# System-browser return-paths beta fixtures

Reviewer-facing fixtures for the beta projection that proves every claimed
identity row defaults to system-browser auth (unless an explicit policy
exception is named), preserves workspace / target / requested-action labels on
return, keeps granted scope from widening requested scope, and exposes a
closed passkey step-up posture when passkey capability is claimed.

The canonical record kind is
`auth_system_browser_return_paths_beta_page_record`. The schema lives at
[`/schemas/auth/system_browser_return_paths_beta.schema.json`](../../../../schemas/auth/system_browser_return_paths_beta.schema.json).
The beta module lives at
[`/crates/aureline-auth/src/system_browser/beta.rs`](../../../../crates/aureline-auth/src/system_browser/beta.rs)
and the live shell consumer at
[`/crates/aureline-shell/src/system_browser_return_paths/mod.rs`](../../../../crates/aureline-shell/src/system_browser_return_paths/mod.rs).

## Files

These JSON files are generated by the headless inspector so they stay a
literal projection of the seeded
`SystemBrowserReturnPathsBetaPage` (no hand-written drift):

```sh
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- page > page.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- rows > rows.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- support-rows > support_rows.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- defects > defects.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- support-export > support_export.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- summary > render_summary.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- drill-widening-scope > drill_widening_authority_scope.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- drill-passkey-no-fallback > drill_passkey_unavailable_without_fallback.json
cargo run -q -p aureline-shell --bin aureline_shell_system_browser_return_paths -- drill-system-browser-not-default > drill_system_browser_not_default_without_exception.json
```

| File | Purpose |
| --- | --- |
| `page.json` | Full beta audit page (rows + support rows + defects + summary). |
| `rows.json` | Live audited rows. |
| `support_rows.json` | Export-safe support rows. |
| `defects.json` | Defect array (empty on the seeded page). |
| `support_export.json` | Support-export wrapper with metadata-safe defect roll-up. |
| `render_summary.json` | Shell rendering summary. |
| `drill_widening_authority_scope.json` | Drill: granted scope widens beyond requested. |
| `drill_passkey_unavailable_without_fallback.json` | Drill: passkey-unavailable posture without a typed fallback retry-path. |
| `drill_system_browser_not_default_without_exception.json` | Drill: row picks a non-`open_system_browser` default while still quoting `system_browser_default_no_exception`. |

## Protected states covered

- `open_system_browser` defaults on managed/provider-linked rows with
  `loopback_port_pinned` origin validation and `must_match_bound_workspace_and_tenant`.
- Passkey-capable step-up offered before write actions (`passkey_capable_offered`)
  with a typed fallback (`resume_after_step_up`).
- Explicit admin-policy exception (`admin_policy_device_code_required`) that
  swaps the default to device-code without silently widening authority.
- Account-free local row (`account_free_local_no_auth_required`) that grants no
  remote scope.

## Failure drills

The three `drill_*.json` fixtures intentionally trip the validator:

- `drill_widening_authority_scope.json` raises
  `return_widens_authority_scope`.
- `drill_passkey_unavailable_without_fallback.json` raises
  `passkey_unavailable_without_honest_fallback`.
- `drill_system_browser_not_default_without_exception.json` raises
  `system_browser_not_default_without_explicit_exception`.

Each drill is replayed end-to-end by the integration test at
[`crates/aureline-shell/tests/system_browser_return_paths_beta_fixtures.rs`](../../../../crates/aureline-shell/tests/system_browser_return_paths_beta_fixtures.rs).
