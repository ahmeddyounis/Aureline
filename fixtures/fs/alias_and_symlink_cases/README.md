# Alias and symlink cases

Compact fixtures for the identity questions that recur across editor,
breadcrumb, review, support-export, and save-target review surfaces:

- which path the user opened;
- which canonical object the next write targets;
- which aliases share dirty-buffer authority; and
- which guarantees weaken on remote, policy-gated, or watch-degraded
  roots.

These files complement `fixtures/fs/save_truth_cases/` rather than
replacing them:

- `save_truth_cases/` describes full save inputs and expected outcomes;
- this directory isolates the identity and disclosure facts later
  surfaces reuse.

Each file carries a stable `case_id` that is registered in
`fixtures/fs/identity_corpus_manifest.yaml`.
