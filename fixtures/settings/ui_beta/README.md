# Settings UI beta fixtures

These fixtures are protected examples for the settings UI beta
projection in `aureline-settings`. They are generated by the
`aureline_settings_inspect` CLI bin so the on-disk JSON stays a
literal projection of the inspector resolver.

| File | What it shows |
|---|---|
| `page_all_settings.json` | Page projection grouped by setting-id prefix with restart, policy, and redaction banners. |
| `inspector_pane_security_egress.json` | Inspector pane for a policy-locked AI egress setting with the source-chain table and lock explanation. |
| `write_composer_policy_denied.json` | Scope-explicit write composer for a user-scope write that is denied by an admin policy; the record carries the owner label and remediation. |
| `support_export_parity.json` | Support export carrying both the beta UI page and the inspector records the rows were built from. UI, CLI, and support exports share `source_record_ref` per setting. |

## Regenerating

```sh
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- ui-beta-page > fixtures/settings/ui_beta/page_all_settings.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- ui-beta-inspector security.ai.egress_policy > fixtures/settings/ui_beta/inspector_pane_security_egress.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- ui-beta-write-composer > fixtures/settings/ui_beta/write_composer_policy_denied.json
cargo run -q -p aureline-settings --bin aureline_settings_inspect -- ui-beta-support-export > fixtures/settings/ui_beta/support_export_parity.json
```

## Validation

The integration test
`crates/aureline-settings/tests/ui_beta_fixtures.rs` replays the
fixtures through the Rust types and checks that:

- the page exposes editor and security groups,
- the policy banner reports at least one locked row sourced from the
  admin policy bundle,
- the restart banner reports at least one row that needs a restart,
- the inspector pane's source-chain table labels the winning row,
- the denied write composer carries the policy owner label, policy
  source ref, and remediation phrasing,
- and the support export's effective-setting records share
  `source_record_ref` with the page rows.
