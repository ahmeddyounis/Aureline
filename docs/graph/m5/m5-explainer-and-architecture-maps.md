# M5 explainer-and-architecture-maps packet

This document describes the canonical packet that carries the **M5 explainer snapshots and
architecture maps** — the honest answer the M5 graph surfaces give to *what is this part of the
workspace, and what backs that explanation?* Where the [workset-scope packet](m5-workset-scope.md)
answers *what slice am I looking at?*, the [topology-identity packet](m5-topology-identity.md)
answers *which exact graph object is this?*, the [impact-query packet](m5-impact-query.md) answers
*is this empty impact answer safe?*, the [ownership-and-contracts packet](m5-ownership-and-contracts.md)
answers *who owns this, and what kind of source said so?*, and the
[graph-governance matrix](m5-graph-governance.md) freezes *which depth claim a lane may publish*,
this packet answers the question onboarding tours, review explainers, the docs/browser surface, and
AI context inspectors all ask of an architecture explanation: **is this generated prose or curated
truth, what files, symbols, docs packs, ADRs, curated notes, and graph objects back it, and how
fresh and confident is it?**

It is the user-facing companion to the governed artifact at
`artifacts/graph/m5/m5-explainer-and-architecture-maps.json` and the typed model in the
`aureline-graph` crate (`m5_explainer_and_architecture_maps`).

## Why generated-versus-curated matters

A fluent architecture explanation is dangerous when its source is hidden: a generated summary read
as curated truth can quietly outrun the citations, scope, and freshness it actually rests on. This
packet refuses to flatten that state. Every [`ExplainerSnapshot`](#snapshots) carries an explicit
[`ExplanationSourceClass`](#source-classes), every snapshot must cite at least one
[`ExplainerCitation`](#citations), and every snapshot carries freshness and confidence cues.

### Source classes

| Class | Meaning | Authority |
| --- | --- | --- |
| `curated` | Curated first-party prose authored or reviewed by a maintainer | authoritative |
| `imported` | Prose imported from a connected doc pack or provider | imported |
| `generated` | Prose generated by a heuristic, summarization, or AI producer — a synthesis, not declared truth | generated |

Source class is a **precedence**: `curated` outranks `imported`, which outranks `generated`. Every
consumer binding **preserves the source-class label** so a generated explanation is never flattened
into curated truth once it leaves the originating panel.

## What this packet covers

Each entry in `snapshots` is an `ExplainerSnapshot` carrying:

- the `subject_id` it explains (in the shared topology identity space) and the `subject_kind`;
- a `title` and a redaction-aware `prose_ref` — never the raw prose body;
- the `source_class`, a `freshness` and `confidence` cue, and a `visibility` scope;
- the `cited_ids` that back it (**at least one** is required); and
- the `navigation_affordances` it offers and the open `follow_up_ids` it surfaces.

### Citations

Each entry in `citations` is an `ExplainerCitation` with a [`CitationKind`](#citation-kinds), a
canonical `target_ref`, a redaction-aware `display_label`, freshness and confidence cues, and an
`export_permalink` that embeds the canonical citation id.

| Kind | Cites |
| --- | --- |
| `file` | A workspace file |
| `symbol` | A code symbol such as a function, type, or method |
| `doc_pack` | A documentation or knowledge pack |
| `adr` | An architecture decision record |
| `curated_note` | A curated maintainer note |
| `graph_object` | A canonical graph node or edge object |

### Generated prose never stands without citations

The headline guardrail: every snapshot — and especially every `generated` one — **must cite at
least one citation** and carry non-empty freshness and confidence cues. A snapshot with no
citations fails validation (`SnapshotMissingCitations`); one with no freshness or confidence cue
fails (`SnapshotMissingFreshnessOrConfidence`).

### Architecture maps are never canvas-only

Each snapshot declares the [`NavigationAffordance`](#navigation-affordances)s it offers. The
`keyboard`, `list_table`, and `screen_reader` paths are **required**, so topology exploration always
has an equivalent accessible path; `canvas` is the optional visual map. A snapshot that offers only
a canvas fails validation (`CanvasOnlyNavigation`).

### Visibility never widens past its scope

Each snapshot declares an [`ExplainerVisibility`](#visibility): `public`, `internal`, or `private`.
Private (policy-scoped) prose is shown only in-product to authorized users and is **never
exported**. Each consumer binding declares a `max_visibility` ceiling, and a binding may not carry a
snapshot more restricted than its ceiling. The support-export binding may carry only export-safe
(`public` or `internal`) snapshots, and the export projection redacts every `private` snapshot
entirely.

### Carried beyond one panel

Each of `onboarding_tour`, `review_explainer`, `docs_browser`, `ai_context_inspector`, and
`support_export` carries exactly one `ExplainerConsumerBinding`, stamped with the active snapshot
and scope, and each binding **preserves source-class labels** so a generated explanation is never
flattened into curated truth downstream. Onboarding, review, docs, and AI all point at the **same**
explainer snapshot rather than regenerating divergent architecture stories. The `support_export`
binding **must carry every export-safe snapshot and no private one**, so support and enterprise
review can inspect the architecture story without a private dashboard lookup or a canvas-only
assumption.

## Guardrails proven

- A snapshot with no citations fails validation (`SnapshotMissingCitations`); one with no freshness
  or confidence cue fails (`SnapshotMissingFreshnessOrConfidence`).
- A snapshot that offers only a canvas — no keyboard, list/table, and screen-reader path — fails
  validation (`CanvasOnlyNavigation`).
- A snapshot or citation permalink that is empty or does not embed its id fails validation
  (`UnsafeSnapshotPermalink`, `UnsafeCitationPermalink`).
- A snapshot that cites an undeclared citation or references an undeclared follow-up fails
  validation (`UnresolvedCitationRef`, `UnresolvedFollowUpRef`).
- A binding that flattens source labels fails validation (`SourceLabelsNotPreserved`); a binding
  that carries a snapshot beyond its visibility ceiling fails (`VisibilityExceedsBinding`).
- A binding not stamped with the active snapshot or scope fails validation
  (`SnapshotBindingMismatch`, `ScopeIdMismatch`); a missing surface binding fails
  (`MissingSurfaceBinding`).
- An export-safe snapshot not carried by the support-export binding fails
  (`ExportSafeSnapshotMissingFromSupportExport`), and a `private` snapshot carried by it fails
  (`PrivateSnapshotInSupportExport`).

## Upstream provenance

The packet binds to the canonical graph-depth governance matrix
(`artifacts/graph/m5/m5-graph-governance.json`), the workset-scope packet
(`artifacts/graph/m5/m5-workset-scope.json`), and the topology-identity packet
(`artifacts/graph/m5/m5-topology-identity.json`) whose node identity space its subjects and
citations reuse.

## Help surface and explainer badge

Help and docs surfaces narrow from this one packet: the **explainer badge** reflects the source
class of an explanation (curated, imported, or generated) alongside its freshness and confidence,
and the help surface explains each citation kind and navigation affordance rather than re-describing
the architecture story by hand.

### Authentication area overview prose

The authentication area (`crates/auth`) owns login, session-token issuance, and rotation. This
curated overview cites the login file and `authenticate` symbol, the auth knowledge pack, ADR 0007
on session-token rotation, and the auth team's curated login-invariants note.

<a id="billing-gateway-prose"></a>

### Billing gateway dependency prose

The billing area depends on an imported `billing-gateway` provider resource. This imported snapshot
cites the gateway file and the `depends_on` graph edge; its freshness is `imported` and confidence
`medium`, and it offers a refresh follow-up.

<a id="auth-login-flow-prose"></a>

### Login authentication flow prose

This generated snapshot summarizes the login authentication flow for the `authenticate` symbol. It
is tagged `generated` with `cached` freshness and `low` confidence, cites the login file and symbol,
and surfaces a request-curated-review follow-up so it can be promoted to curated truth.
