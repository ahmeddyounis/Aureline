# Runtime Continuity Surface Qualification

Runtime-continuity, restore, and terminal-governance posture is quoted from:

[`artifacts/runtime/runtime-continuity-surface-qualification/support_export.json`](../../artifacts/runtime/runtime-continuity-surface-qualification/support_export.json)

Use each profile row directly:

- `displayed_label` is the exact lifecycle/support label docs/help, Help /
  About, support, and public-truth consumers render.
- `narrow_reasons` explains why a profile narrowed instead of inheriting generic
  desktop continuity.
- `packet_refs` shows which checked runtime packet, protocol corpus, and restore
  drill currently back the row.

The checked packet keeps `local_only`, `mirrored`, and `managed` at `stable`.
`browser_handoff` is intentionally `preview` until a checked browser-handoff
runtime packet proves live terminal authority, clipboard-write posture, and
restore-no-rerun semantics without leaning on desktop evidence.
