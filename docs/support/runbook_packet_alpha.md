# Runbook Packet Alpha

This document describes how incident workspaces consume support runbook
packets and export them through redacted support bundles.

## Source of truth

The packet implementation lives in `crates/aureline-incident`.
Runbook packet shape remains owned by:

- `docs/support/runbook_execution_contract.md`
- `schemas/support/runbook_packet.schema.json`
- `schemas/support/runbook_step_result.schema.json`

The incident crate parses the support runbook packet only far enough to
preserve the join points needed by support export: packet id, runbook id,
version, source document, freshness state, rollback posture, step counts,
policy refs, redaction class, and exact-build refs.

## Export rules

An incident runbook packet is exportable only as redacted metadata:

- exact-build refs must be present;
- source document and docs freshness state must travel with the packet;
- rollback posture and mutating-step count stay visible;
- raw command lines, provider URLs, payloads, approval bodies, terminal
  transcripts, and secrets remain excluded;
- support-bundle redaction controls are generated by
  `SupportBundlePreviewBuilder`, not by incident-local rules.

## Missing-span honesty

Incident workspaces can be useful before every span is available. The
alpha packet therefore records typed gaps instead of pretending the
chronology is complete.

Missing spans cover:

- trace or log windows unavailable at export time;
- crash symbolication reports not produced yet;
- provider callbacks or handoff events that never arrived;
- support-bundle manifest refs that could not be joined.

When any required span is missing, `span_coverage.complete_coverage_claimed`
is `false`, and the redacted export preview includes the
`support.item.incident.missing_spans` row with an honesty marker.

## Local continuity

Provider or hosted lanes can be `degraded`, `unavailable`, or
`not_configured` while the workspace still remains useful. Local review
continues when `local_continuity_state` permits it, and the exported
support preview remains reopenable without network access.
