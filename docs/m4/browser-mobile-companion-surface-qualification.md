# Browser/Mobile Companion Surface Qualification

The stable release qualifies browser and mobile companion surfaces only where the checked-in packet proves narrow scope, visible freshness, authority boundaries, and exact desktop handoff.

Stable companion rows:

- Browser companion review, comment, and bounded light edit.
- Mobile notification triage and handoff card.
- Exact-target desktop handoff and placeholder reopen.

Preview companion rows:

- Browser docs context.
- Browser companion session follow.
- Mobile companion session join and rejoin.

Companion surfaces must not be described as available everywhere or desktop-equivalent. Use the packet vocabulary: `review`, `docs`, `follow`, `join`, `light_edit`, `notification_triage`, and `handoff_only`. Unsupported and desktop-authoritative actions must be shown before the user attempts them.
